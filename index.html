wasd -- move the cursor

Arrows / ijkl -- move the camera

Page Up / u -- zoom in

Page Down / o -- zoom out

click on texture sheet -- set the default palette slot to that texture

space / 0 -- paint the cell under the cursor with the default palette slot

1-9 -- paint the cell under the cursor with the numbered palette slot

f -- toggle palette set mode

g -- toggle North South palette set mode

h -- toggle East West palette set mode

1-9 pallette set mode -- change the numbered palette slot to match the default slot

p -- print the map data to the textbox

[ -- open the map data from the textbox, if no data is entered it will create a blank map with specified dimensions

m -- mirror the default pallette on the EW sides

n -- mirror the defualt pallette on the NS sides

t -- toggle weather a texture is partially see through, currently broken in the engine

Textures are oriented relative to the world, not a cell.

This means that if from the north a detail is on the right, from the south it will be on the left.

In the top down map, cells with multiple textures are shown with the NS texture on the left and the EW texture on the right.

There is support for portals in the engine but they are not yet supported by the map editor.

Portals are defined with the portalNS and portalEW cell properties, each of which should be an object with the destination x and y if declared. Example: {portalNS:{x:2,y:2},portalEW:{x:7,y:2}}
